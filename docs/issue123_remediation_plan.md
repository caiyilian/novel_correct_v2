## 2. 执行原则

- 每个子阶段单独 issue、单独分支、单独 PR、单独验收。
- 每个阶段只修一个卷或一个明确问题，不再把多个大问题合在一起。
- 所有修复必须落到最终主产物 `output/corrected_第X卷.txt`，不能只停留在 stage 中间文件。
- 答案文件只用于结构验收、缺口定位和计数目标，不用于复制正文。
- Ollama 只用于小候选裁决，输出 JSON 决策；Python 负责 offset patch、校验和报告。
- 每个阶段验收至少包含：
  - `tools/final_audit.py` 对相关卷的结果；
  - 该卷 `compare_dialogues.py` 报告；
  - 该卷 `verify_against_answer.py` 报告；
  - 该阶段 apply / decision / audit 报告；
  - 明确说明是否还有剩余缺口。
